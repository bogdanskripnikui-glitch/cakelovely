# CakeLovely Sync

Первый рабочий каркас облачного проекта для учета продаж бенто-тортов, капкейков, трайфлов и наборов.

## Что уже есть

- структура данных по городам и месяцам
- ручные цены по категориям
- ручные количества по категориям
- коэффициенты, считающиеся от цены бенто-стандарта
- расчет веса продаж и выручки
- стартовый интерфейс на Next.js
- локальное сохранение в браузере
- заготовка подключения Supabase для синхронизации

## Логика

- `Бенто Standart = 1`
- остальные категории считаются относительно его цены в конкретном месяце
- для каждого города и месяца хранятся свои цены и количества
- синхронизация в будущем будет идти через Supabase
- при наличии ключей Supabase можно будет перенести сохранение в облако и открыть тот же аккаунт с другого устройства

## Старт

```bash
npm install
npm run dev
```

## Переменные для облака

Если будем подключать синхронизацию, понадобятся:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

## Дальше

1. Подключить Supabase проект.
2. Сделать авторизацию под одним аккаунтом.
3. Перенести сохранение записей в облако.
4. Подключить редактирование месячных данных в интерфейсе.
